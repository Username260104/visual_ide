# VIDE Phase 8 Manual Smoke Checklist

작성일: 2026-03-26

목적:
- 자동화 환경과 분리해서, 사람이 5분 안에 Phase 8 마감 여부를 확인한다.
- 구현 완료 여부가 아니라 `실사용 흐름이 실제 UI에서 자연스럽게 이어지는지`를 본다.

준비:
- 로컬 서버 실행
- 테스트용 프로젝트 1개
- Direction 1개 이상
- 이미지 노드 1개 이상

공통 예시 텍스트:
- 프로젝트 브리프: `프리미엄 스킨케어 런칭 비주얼. 임상적 신뢰감은 유지하되 차갑지 않게 보일 것.`
- 브랜드 톤: `정제됨, 프리미엄, 깨끗함, 과임상 금지`
- 타깃 오디언스: `30대 초중반 여성, 프리미엄 뷰티 관심층`
- 제약 조건: `카피 공간 필요, 병 실루엣 명확, 과한 식물 연출 금지`
- 방향 가설: `미니멀 스튜디오 톤이 브랜드 신뢰감과 전개 확장성에 가장 잘 맞는다.`
- 적합 기준: `제품 존재감이 강하고, 카피 공간이 확보되며, 런칭 키비주얼로 확장 가능해야 한다.`
- 피해야 할 느낌: `과한 자연주의, 너무 차가운 병원 광고 느낌, 배경 과밀`
- 참고 메모: `아크릴 받침, 차분한 하이라이트, 중앙 구도 선호`
- Generate 프롬프트 예시: `premium skincare bottle on clear acrylic, clean studio light, minimal gradient background, centered composition, ample copy space`
- 채택 이유 예시: `제품 존재감과 카피 공간의 균형이 가장 좋고 브랜드 톤과도 가장 잘 맞는다.`

## 1. Settings 전략 저장 확인

순서:
- 프로젝트 화면에서 `Settings` 탭으로 이동
- 아래 예시 문구를 입력
- blur 또는 `Ctrl/Cmd+Enter`로 저장

입력 예시:
- 프로젝트 브리프: `프리미엄 스킨케어 런칭 비주얼. 임상적 신뢰감은 유지하되 차갑지 않게 보일 것.`
- 브랜드 톤: `정제됨, 프리미엄, 깨끗함, 과임상 금지`
- 타깃 오디언스: `30대 초중반 여성, 프리미엄 뷰티 관심층`
- 제약 조건: `카피 공간 필요, 병 실루엣 명확, 과한 식물 연출 금지`
- 방향 가설: `미니멀 스튜디오 톤이 브랜드 신뢰감과 전개 확장성에 가장 잘 맞는다.`
- 적합 기준: `제품 존재감이 강하고, 카피 공간이 확보되며, 런칭 키비주얼로 확장 가능해야 한다.`
- 피해야 할 느낌: `과한 자연주의, 너무 차가운 병원 광고 느낌, 배경 과밀`
- 참고 메모: `아크릴 받침, 차분한 하이라이트, 중앙 구도 선호`

기대 결과:
- 저장 중/저장 완료 피드백이 보인다
- 새로고침 후에도 값이 유지된다
- 최근 활동에 전략 업데이트 이벤트가 보인다

## 2. GenerateDialog 컨텍스트 노출 확인

순서:
- `Directions` 탭으로 이동
- `이미지 생성` 클릭

기대 결과:
- `프로젝트 컨텍스트` 카드가 보인다
- 방금 저장한 프로젝트 전략 텍스트가 그대로 보인다
- 빈 상태가 아니라 실제 전략이 surface 된다

## 3. Staging Tray 생성/채택 확인

순서:
- GenerateDialog에 아래 예시 프롬프트 입력
- 이미지 생성
- staging 결과가 하단 tray에 올라오는지 확인
- 후보 1개 이상 선택
- `비교 후 채택` 클릭
- 채택 이유 입력 후 확정

입력 예시:
- 프롬프트: `premium skincare bottle on clear acrylic, clean studio light, minimal gradient background, centered composition, ample copy space`
- 채택 이유: `제품 존재감과 카피 공간의 균형이 가장 좋고 브랜드 톤과도 가장 잘 맞는다.`

기대 결과:
- 생성 결과가 즉시 노드로 들어오지 않고 staging에 먼저 쌓인다
- 비교 모달에서 채택 후보와 기각 후보가 나뉘어 보인다
- 채택 후에만 캔버스에 노드가 생긴다

## 4. DetailPanel 전략 요약 확인

순서:
- 방금 채택된 노드를 선택

기대 결과:
- DetailPanel에 `방향 전략` 카드가 보인다
- `방향 가설`, `적합 기준`, `피해야 할 느낌`, `참고 메모`가 읽기 전용으로 보인다
- `수동 기록` 영역이 함께 보인다

## 5. VariationPanel 컨텍스트 확인

순서:
- DetailPanel에서 `변형 만들기` 진입

추가 입력 예시:
- 의도 태그 예시: `톤 조정`
- 변경 요소 예시: `조명`
- 메모 예시: `조명을 조금 더 부드럽게 하고 제품 윤곽은 유지해 주세요.`

기대 결과:
- `방향 전략` 카드가 보인다
- `프로젝트 컨텍스트` 카드가 보인다
- 전략 텍스트를 보면서 변형 입력을 작성할 수 있다

## 6. Timeline 기록 확인

순서:
- 다시 `Settings` 탭으로 이동
- `최근 활동` 확인

기대 결과:
- 전략 업데이트 이벤트가 보인다
- 비교/채택 흐름 이후 `비교` 이벤트가 보인다
- 노드 선택 기준의 상세 타임라인에서도 관련 기록이 이어진다

## 판정 기준

아래 6개가 모두 `통과`면 R5를 완료로 본다.

- Settings 전략 저장
- GenerateDialog 컨텍스트 노출
- staging 생성/채택
- DetailPanel 전략 요약
- VariationPanel 컨텍스트
- Timeline 이벤트

## 비고

- `storage orphan cleanup`은 이 체크리스트에서 직접 재현하지 않는다.
- 그 항목은 코드 반영과 build 검증 완료 상태로 관리한다.
